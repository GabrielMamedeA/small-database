# Book Database in C

A simple book management system in C using **binary files** for storage.  
Supports **CRUD operations** (Create, Read, Update, Delete) and searching and sorting by all book attributes.

## Features

- Add new books with automatic ID assignment
- List all books
- Search books by ID, title, author, theme, read status, release year and your own custom filters
- Update book information
- Soft delete books (mark as inactive)
- Persistent storage using binary files (`books.dat` and `last_id.dat`)
- Import data from `.CSV` files and export to `.TXT`
- Sort your books by any attribute (`quicksort`)
- **Database statistics** — real-time summary of the entire collection
- **LZW Compression & Decompression** — lossless binary compression of the database file

## Book Structure

Each book contains the following attributes:

```c
typedef struct {
    unsigned short id;
    char title[80];
    char author[50];
    char theme[30];
    bool read;
    unsigned short releaseYear;
    bool active;
} Book;
```

## Feature Details

### 📊 Database Statistics (`getDatabaseStatistics`)

Performs a single pass over `books.dat` and displays a formatted dashboard including:

- Total, active, and inactive book counts (with percentages)
- Read vs. unread ratio
- Number of unique authors and themes
- Oldest and newest book by release year
- Most frequent author and theme (with book counts)
- Database file size in bytes

### 🗜️ LZW Compression (`compressDatabase` / `decompressDatabase`)

Implements the **Lempel–Ziv–Welch (LZW)** algorithm directly on the binary database file:

- **12-bit codes** — dictionary supports up to 4,096 entries
- **Compression** reads `data/books.dat` and writes `data/books.lzw`, which includes a 4-byte magic header (`LZW\x01`) and the original file size for integrity validation
- **Decompression** reads `data/books.lzw`, validates the header, and fully reconstructs `data/books.dat`
- Codes are packed as **two 12-bit codes per 3 bytes** for space efficiency
- A hash table is used on the encoder side for fast O(1) dictionary lookups
